# 精读模式 — Book Reading

## 目标

生成一份可以独立阅读、适合复盘和收藏的系统读书笔记。它不是逐章摘抄，而是重构全书的问题、观点、证据、结构和启发。

保存到：

```text
book-notes/<name>/book-reading.md
```

## 固定结构

```markdown
# <书名> — 精读笔记

> **书名**：...
>
> **作者**：...
>
> **版本/来源**：...
>
> **阅读对象**：...
>
> **阅读难度**：...

## TL;DR

## 全书问题链

## 作者的核心主张

## 章节地图

## 关键概念与术语

## 重要论证与案例

## 可复用的方法或模型

## 我不同意或需要警惕的地方

## 和其他书/观点的关系

## 深度问答

## 行动清单

## 摘录与定位
```

## 写作要求

- `TL;DR` 用 150-250 字回答：这本书为什么写、主张什么、读者能带走什么。
- `章节地图` 说明每章在全书中的功能，避免流水账。
- `关键概念与术语` 对外文术语给中文解释，中文解释不超过 10 字。
- `重要论证与案例` 要写清作者如何支撑观点。
- `需要警惕的地方` 不能空泛，要指出适用边界、证据薄弱点或时代局限。
- `深度问答` 写 6-8 个高价值问题，每个回答 2-4 句话。
- `行动清单` 要可执行，避免“提高认知”这类空话。
- 尽量保留章节、页码或位置线索。
